# AIoT 认证测试平台指南

认证测试平台是小米 AIoT 测试部开发的新一代认证测试服务平台，面向 IoT 开发者，集成了多种通信能力和测试工具，助力开发者高效自测、高质上线。

## 平台通信能力

### Wi-Fi 模组产品

Wi-Fi 接入产品支持两种通信方式：

- **云端通信**：平台已实现，可向米家云服务发送控制命令到设备
- **局域网通信**：研发中，预计 2024 年随局域网测试用例集成

### BLE/BLE Mesh 模组产品

BLE/BLE Mesh 产品只支持与蓝牙或 Mesh 网关通信。平台支持连接小米中枢网关，通过中枢网关与 BLE/BLE Mesh 产品进行通信。

## OTA 测试工具

用于设备升/降级压力和遍历测试，支持 `WiFi`、`蓝牙 Mesh`、`PLC` 类型设备。

### WiFi/PLC 设备功能

| 功能 | 说明 |
| --- | --- |
| 压力测试 | 选择两个固件版本和测试次数，在版本之间进行指定次数的 OTA 升降级测试 |
| 遍历测试 | 选择起始版本（非最新），依次将设备从各版本升级到最新版本 |
| 固件刷包 | 通过 RPC 命令给设备刷入指定固件包，弥补烧录方式刷包的不足 |
| 自动升级 | 验证 MCU 版本的自动升级功能是否正常 |

### 蓝牙设备功能

Android 手机下载蓝牙 OTA 升降级测试工具 APK，进行 OTA 升降级及压力测试。蓝牙设备需通过工具先下载固件版本包，再通过蓝牙传输给设备进行升级。

### 使用步骤

1. 准备小米 IoT 开发者平台账号和被测设备
2. 将被测设备绑定到开发者账号上
3. 确保设备属于该开发者所在的企业组
4. 确保设备为在线状态
5. 使用开发者账号登录客户端
6. 选择测试环境，找到待测设备
7. 指定 OTA 压力测试次数、固件升降级版本号，点击"开始测试"

### 验收标准

- 压力测试要求 OTA 次数为 100 次
- 警告次数比例不大于 10%
- 失败次数为 0

## SPEC 测试工具

用于验证产品实现是否符合 instance 描述，确保产品和 instance 的一致性。

### 功能特性

- 根据 instance 自动生成测试用例集，遍历检查各种 `properties` 和 `actions`
- 支持固件 RPC 耗时测试
- 支持设备调试

### 使用步骤

1. 准备小米 IoT 开发者平台账号和被测设备
2. 将被测设备绑定到开发者账号上（不要使用共享设备，否则会导致 notify 测试失败）
3. 确保设备属于该开发者所在的企业组且为在线状态
4. 选择设备测试环境和待测设备，自动生成测试用例集
5. 可调整测试集（如处理互斥属性）并存档供以后使用
6. 点击开始测试，实时显示测试进度
7. 测试完成后可选择是否上传测试报告

### 支持耗时统计的模组

| 模组型号 | 芯片平台 | 最低支持版本 |
| --- | --- | --- |
| ESP-WROOM-32D/ESP32-WROOM-32U | ESP32 | 2.0.8 |
| ESP-WROOM-02D/ESPWROOM-02U | ESP8266 | 2.0.5 |
| MHCWB2P | MTK7697 | 2.0.3 |
| MHCWB4P-B/MHCWB4P-IB | 8720c | 2.0.1 |
| MHCWB6P-IB | esp32-c3 | 所有版本 |
| OT-Client SDK(Linux) | NA | 4.0.7 |
| OT-Client SDK(Android) | NA | 4.2.1 |

### 设备调试

选择设备测试环境和待测设备后，针对每个用例可单独进行测试，支持读缓存、写入、读设备等操作。

## SPEC 协议规范工具

用于检测设备真实功能与开发者平台配置的 SPEC 定义是否一致，帮助快速发现配置与实际功能的差异。

### 实现原理

工具通过设备所属环境和设备 DID 查询设备上报日志，清洗出属性值，并与 SPEC 定义的属性进行对比。系统会识别出 SPEC 中已定义但设备日志中缺失的属性，标记为 `miss`。

### 使用方法

1. 选择设备所属的测试环境
2. 从对应环境中选择待测试设备（最多支持 5 个设备）
3. 设置日志对比时间范围（最长 7 天）
4. 点击"开始测试"按钮启动对比任务

### 结果状态说明

| 状态 | 说明 |
| --- | --- |
| 成功 | 日志信息中找到了匹配的 IID |
| miss | Spec 定义了但实际日志数据中未找到 |
| 确认后通过 | 人工审核后确认该 MISS 属于正常情况 |

## BLE 设备发包规范

### 未配网广播发包测试

**测试准备**：将待测设备置为未配网状态

**操作说明**：PC 工具选中待测设备，设置好未配网广播发送时长，点击开始测试后立即将待测设备断电重启，测试预计持续 33 分钟。

**错误排查与修改建议**：

1. 设备必须有重置按键或其他重置方式，且 `HAVE_RESET_BUTTON` 宏设置为 1
2. 未配网广播周期设置为 100~500ms（推荐 200ms 内）
3. 未配网广播时长设置为 30min 以内

### 其他测试类型

- **确认广播发包测试**：测试设备在配网确认阶段的广播发包规范
- **直连广播发包测试**：测试设备在直连模式下的广播发包规范
- **事件广播发包测试**：测试设备在事件上报时的广播发包规范

## Mesh 设备发包规范

### 未配网广播发包测试

**测试准备**：将待测设备置为未配网状态

**操作说明**：PC 工具选中待测设备，点击开始测试后立即将待测设备断电重启，测试预计持续 33 分钟。

**错误排查与修改建议**：

1. 未配网广播周期设置为 100ms
2. 未配网广播时长设置为 30min
3. 检查周围是否有其他 2.4G 设备干扰，将设备放置在干净环境测试

### 其他测试类型

- **上电广播发包测试**：测试设备上电时的广播发包规范
- **待机广播发包测试**：测试设备待机状态下的广播发包规范
- **控制广播发包测试**：测试设备控制操作时的广播发包规范

## Mesh 设备控制回传成功率测试

测试 Mesh 设备控制命令的回传成功率。

## MCU 验收工具

用于 MCU 串口协议测试，验证 MCU 与模组之间的通信协议是否正确。

## 隐私自动化测试工具

为提升生态链企业针对海外 IoT 产品隐私自测质量，减少隐私测试资源和成本投入而开发。

### 测试范围

目前仅针对接入米家 APP 的海外 IoT 设备。

### 使用步骤

1. **权限申请**：首次使用时点击"申请权限"选项，填写并提交信息
2. **填写产品 model**
3. **选择插件版本**：无法选择的插件版本说明未配置隐私文件
4. **勾选销售国家**：根据产品需求定制表中销售国家进行勾选
5. **选择隐私版本**：按销售地区对应的隐私标准进行选择
6. **上传隐私检测文件**：准备多语产品名称表（excel 格式）
7. **配置链接检测**：将检测链接及出现次数填入工具
8. **开始测试**：点击开始按钮执行测试
9. **查看报告**：测试完成后可选择确认上传，供下次查看

### 注意事项

- 上传的隐私文件需与模板一致，包含"多语言产品"和"撤销授权路径"两张表格
- 语言名称需严格与模板一致（如美式英语、香港繁体、台湾繁体等）
- 电话检测需严格与检测文档中一致

## 测试报告功能

测试结果支持多维度筛选，可根据以下条件查询：

- 产品 model
- 固件版本
- 测试用例
- 测试日期

### 报告内容

- 展示测试重点指标及验收标准（如成功率、延迟等）
- 详细说明测试失败的原因
- 提供可能的修改、优化建议
- 支持下载报告功能
