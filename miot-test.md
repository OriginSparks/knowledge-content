# AIoT 认证测试平台

认证测试平台（以下简称"平台"）是小米 AIoT 测试部开发的全新一代认证测试服务平台。平台专门面向 IoT 开发者研发，集成了多种通信能力和丰富的测试工具。全新的平台将助力开发者**高效自测、高质上线**。

## 通信能力

### 使用 Wi-Fi 模组的产品

Wi-Fi 接入的产品一般支持两种通信方式：**云端通信**和**局域网通信**。平台实现了云端通信的能力，可以和米家插件一样向米家云服务发送控制命令到设备。

> **注意**：局域网通信能力还在研发中，预计在 2024 年随局域网测试用例一同集成。

### 使用 BLE/BLE Mesh 模组的产品

BLE/BLE Mesh 接入的产品只支持和蓝牙或 Mesh 网关进行通信。平台支持连接小米中枢网关，因此可以通过中枢网关与 BLE/BLE Mesh 产品进行通信。

## 测试工具

测试工具基于提测资料中的通用用例进行开发，工具按照测试用例标题进行排列并按照设备接入方式进行归类，工具展示更加直观。开发者根据通用用例 excel 表能够更快速的找到对应测试工具。

**主要特点**：
- 一个测试工具只针对一个测试用例
- 提供详细的使用指引和验收标准
- 开发者可以按照提示一步一步完成测试任务
- 优化操作逻辑，减少开发者自测周期

### OTA 测试工具

用于小米 IoT 设备升/降级压力和遍历测试，支持 WiFi、蓝牙 Mesh、PLC 类型的设备。

**功能介绍**：

**1. WiFi 设备/PLC 设备**

- **压力测试**：用户选择任意两个固件版本和 OTA 测试次数，工具会在两个版本之间进行指定次数的 OTA 升降级测试。通过调用米家给设备发送 RPC 命令的接口，发送 OTA 命令给设备，然后查询设备固件版本信息判断 OTA 是否成功。

- **遍历测试**：用户选择任意一个固件起始版本（非最新），工具会筛选出起始版本及以上版本，并依次将设备从各个版本升级到最新版本。

- **固件刷包**：用于固件版本迭代开发调试过程中，通过发送 RPC 命令的方式给设备刷入指定的固件包，弥补烧录方式刷包的不足。

- **自动升级**：验证新开发支持自动升级功能的 MCU 版本其自动升级功能是否正常。

**2. 蓝牙设备（BLE/BLE-Mesh）**

Android 手机下载安装蓝牙 OTA 升降级测试工具，进行 OTA 升降级以及压力测试。针对通过蓝牙方式联网接入的设备，需要通过工具先下载固件版本包然后再通过蓝牙传输给设备来进行 OTA 升级。

**使用步骤**：
1. 准备小米 IoT 开发者平台账号和被测设备
2. 将被测设备绑定到开发者账号上
3. 确保设备属于该开发者所在的企业组
4. 确保设备为在线状态
5. 使用开发者账号登录客户端
6. 选择测试环境，找到待测设备
7. 指定 OTA 压力测试次数、固件升降级版本号，点击"开始测试"

**验收标准**：
- 原则上测试类型为压力测试，要求 OTA 次数为 100 次
- 警告次数比例不大于 10%
- 失败次数为 0

### SPEC 测试工具

用于验证产品实现是否符合 instance 描述，确保产品和 instance 的一致性。

**功能介绍**：
1. 工具会根据 instance 自动生成测试用例集，对定义的各种 properties 和 actions 进行遍历检查
2. 支持固件 RPC 耗时测试
3. 支持设备调试

**使用步骤**：
1. 准备小米 IoT 开发者平台账号和被测设备
2. 将被测设备绑定到开发者账号上（请不要使用共享的设备进行测试，否则会导致 notify 测试失败）
3. 确保设备属于该开发者所在的企业组
4. 确保设备为在线状态
5. 选择设备测试环境和待测设备后，会自动生成测试用例集
6. 可调整测试集（如处理互斥属性）
7. 可存档测试集供以后使用
8. 点击开始测试，实时显示测试进度
9. 测试完成后可选择是否上传测试报告

**耗时测试**：支持耗时统计的模组包括 ESP32、ESP8266、MTK7697、8720c、esp32-c3、OT-Client SDK(Linux/Android) 等。

**设备调试**：选择设备测试环境、待测设备后，针对每个用例可单独进行测试，支持读缓存、写入、读设备等操作。

### SPEC 协议规范

用于检测设备的真实功能与在开发者平台配置的 SPEC 定义是否一致，帮助开发者快速发现配置与实际功能之间的差异。

**实现原理**：工具通过设备所属环境和设备 DID 查询设备上报日志，清洗出属性值，并与 SPEC 定义的属性进行对比。系统会识别出 SPEC 中已定义但设备日志中缺失的属性，这类属性将被标记为 **miss**（缺失）。

**工具使用**：
1. 选择环境：选择设备所属的测试环境
2. 选择设备：从对应环境中选择待测试设备，最多支持 5 个设备
3. 设置对比时间段：选择日志对比的时间范围，最长 7 天
4. 开始测试：点击"开始测试"按钮启动对比任务

**结果状态**：
- **成功**：日志信息中找到了匹配的 IID
- **miss**：Spec 定义了但实际日志数据中未找到
- **确认后通过**：人工审核后确认该 MISS 属于正常情况

### BLE 设备发包规范

#### 未配网广播发包测试

**测试准备**：将待测设备置为未配网状态

**操作说明**：PC 工具选中待测设备，用户需要根据设备实际情况设置好未配网广播发送时长，点击开始测试后立即将待测设备断电重启，此项测试预计持续 33 分钟。

**错误排查与修改建议**：
1. 设备必须有重置按键或其他重置方式，且 HAVE_RESET_BUTTON 宏设置为 1
2. 未配网广播周期设置为 100~500ms（推荐 200ms 内），miio_ble_user_adv_start(100)
3. 未配网广播时长设置为 30min 以内，在 KEY_NOT_FOUND 回调中设置 miio_system_set_adv_timeout(1800000)

#### 确认广播发包测试
测试设备在配网确认阶段的广播发包规范。

#### 直连广播发包测试
测试设备在直连模式下的广播发包规范。

#### 事件广播发包测试
测试设备在事件上报时的广播发包规范。

### Mesh 设备发包规范

#### 未配网广播发包测试

**测试准备**：将待测设备置为未配网状态

**操作说明**：PC 工具选中待测设备，点击开始测试后立即将待测设备断电重启，此项测试预计持续 33 分钟。

**错误排查与修改建议**：
1. 未配网广播周期设置为 100ms
2. 未配网广播时长设置为 30min，在 UNPROV 回调中设置 miio_system_set_adv_timeout(1800000)
3. 检查周围是否有其他 2.4G 设备干扰，将设备放置在干净环境测试

#### 上电广播发包测试
测试设备上电时的广播发包规范。

#### 待机广播发包测试
测试设备待机状态下的广播发包规范。

#### 控制广播发包测试
测试设备控制操作时的广播发包规范。

### Mesh 设备控制回传成功率测试
测试 Mesh 设备控制命令的回传成功率。

### MCU 验收工具
用于 MCU 串口协议测试，验证 MCU 与模组之间的通信协议是否正确。

### 隐私自动化测试工具

为提升生态链企业针对海外 IoT 产品隐私自测质量，减少隐私测试资源和成本的投入而开发。

**测试范围**：目前仅针对接入米家 APP 的海外 IoT 设备。

**使用步骤**：
1. **权限申请**：首次使用工具时，点击隐私工具中的"申请权限"选项，填写并提交相应信息
2. **填写产品 model**
3. **选择插件版本**：如果不能选择某插件版本，说明该插件版本未配置隐私文件
4. **勾选销售国家**：根据该产品的需求定制表中销售国家进行勾选
5. **选择隐私版本**：如果使用最新版本隐私模版的产品，也仍需按照销售地区对应的隐私标准进行选择
6. **上传隐私检测文件**：准备多语产品名称表，格式为 excel 文件
7. **配置链接检测**：将每个检测的链接及在文件中出现的次数填在工具内
8. **开始测试**：点击开始按钮执行测试
9. **查看报告**：测试完成后可选择确认上传，下次仍可以查看

**注意事项**：
- 上传的隐私文件需要同模板一致，有"多语言产品"和"撤销授权路径"两张表格
- 语言名称需严格同模板一致，如美式英语、香港繁体、台湾繁体等
- 电话检测需要严格与检测文档中一致

## 测试报告

测试结果支持多维度筛选，可以根据以下条件进行查询：
- 产品 model
- 固件版本
- 测试用例
- 测试日期

用户在输入 model 和测试日期的情况下可以了解产品在所选时间段内所有用例的测试情况，全面把握测试进度。

**测试报告内容**：
- 展示测试的重点指标及验收标准（如成功率、延迟等）
- 详细说明测试失败的原因
- 提供可能的修改、优化建议
- 支持下载报告功能
